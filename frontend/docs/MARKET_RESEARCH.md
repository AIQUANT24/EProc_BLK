## 2. PPP-MII Policy Framework

### Policy Evolution

| Date | Event |
|------|-------|
| June 2017 | PPP-MII Order issued by DPIIT |
| September 2020 | Major revision — tightened DVA rules |
| July 2024 | Latest revision — clarity on local content calculation |
| January 2025 | CSIR compliance instructions issued |
| 2025 | MeitY guidelines on PPP-MII framework published |
| 2025 | DPIIT proposes third-party tender compliance vetting |

### Key PPP-MII Provisions (Revised July 2024)

1. **Local Content Definition:** Value added in India, excluding imported items
2. **Class I Supplier:** ≥50% local content (full purchase preference)
3. **Class II Supplier:** ≥20% local content (preference if no Class I available)
4. **Non-Local:** <20% local content (lowest priority)
5. **Verification:** Self-certification + government can verify via statutory auditor
6. **Sector Rules:** Sector-specific DVA thresholds set by nodal ministries
7. **Penalties:** False declaration → debarment + 2% contract value penalty

### Compliance Challenges

| Challenge | Impact |
|-----------|--------|
| Self-certification without verification | High fraud risk |
| No centralized DVA calculation system | Inconsistent application |
| Manual verification by statutory auditors | Slow, expensive, not scalable |
| No cross-referencing with customs/GST data | Easy to falsify origin claims |
| Lack of component-level tracking | Aggregate claims hide imported content |
| No blockchain/immutable audit trail | Records can be tampered |

## 6. International Benchmarks

| Country | System | Comparison |
|---------|--------|------------|
| **USA** — Buy American Act | SAM.gov, FPDS | Product-level, not component-level |
| **EU** — Public Procurement Directives | TED (Tenders Electronic Daily) | No domestic content verification |
| **South Korea** — Korea ON-line E-Procurement | KONEPS | Digital but no DVA equivalent |
| **Australia** — Commonwealth Procurement Rules | AusTender | Guidelines, no automated verification |

> India's PPP-MII is **more granular** than most international equivalents, creating a unique need for NMICOV's component-level verification.
